API Keys
GET /{collection} List documents with filters, sorting and pagination ▼
max 500
Operators: == != < <= > >= in not-in array-contains array-contains-any  |  For in/not-in: value1,value2
GET /{collection}/count Count documents (supports where filters) ▼
GET /{collection}/{id} Get a single document by ID ▼
HEAD /{collection}/{id} Check if a document exists (200 = yes, 404 = no) ▼
GET /{collection}/_schema Infer field names and types from the first 20 documents ▼
GET /{collection}/{id}/{subcollection} List documents inside a subcollection ▼
POST /{collection} Insert a single document (auto-generated ID) ▼
POST /{collection}/_batch Insert multiple documents in one request (max 500) ▼
PUT /{collection}/{id} Replace an existing document entirely ▼
PUT /{collection}/{id}?upsert=true Create document if it doesn't exist, replace if it does ▼
PATCH /{collection}/{id} Partially update a document ▼
PATCH /{collection}?where=field:op:value Update all documents matching a filter (max 500) ▼
At least one filter is required
DELETE /{collection}/{id} Delete a single document ▼
DELETE /{collection}/_batch Delete multiple documents by ID array (max 500) ▼
DELETE /{collection}?confirm=true Delete ALL documents in a collection ▼
⚠️ Destructive. Deletes up to 500 documents per call. Call repeatedly until empty.
